MISSION LIST
M-01
ワールド改変ってなに？
PENDING
▼

VRChatには「ワールド」と呼ばれる3D空間がある。誰かが作ったワールドのデータをもらって、自分好みに変えてアップロードすること——それが「ワールド改変」。

改変には元データ（unitypackage）の配布が必要。必ず利用規約を確認しよう
改変したワールドはVRChatに自分のワールドとして公開できる
ゼロから作るより簡単・速い。最初の練習に最適
M-02
必要なものを揃えよう
PENDING
▼

改変に必要なソフトは3つだけ。全部無料で使えるが、インストール前に必ずバージョンを確認すること。

1Unity：VRChat公式が指定するバージョンを使う（Unity Hubで管理）
2VRChat Creator Companion（VCC）：公式ツール。SDKの管理はここで行う
3改変したいワールドのデータ：配布元からunitypackageをダウンロード
⚠ VRChat公式サイトで「推奨Unityバージョン」を必ず確認。最新版が正解とは限らない。
M-03
VCCでプロジェクトを作ろう
PENDING
▼

VRChat Creator Companion（VCC）を使って、Unityプロジェクトを作成する。直接Unityを開くより、VCCを通す方がSDKが自動で入って楽。

1VCCを開いて「Create New Project」→「World」テンプレートを選ぶ
2プロジェクト名はわかりやすい名前に（日本語は避けるのが無難）
3作成後「Open Project」でUnityが自動で開く
⚠ Unityの初回起動は数分かかることがある。フリーズではないので待とう。
M-04
ワールドデータを読み込もう
PENDING
▼

ダウンロードしたワールドのunitypackageをプロジェクトに読み込む。ここからようやく改変の準備が整う。

1Unityが開いた状態で、unitypackageファイルをダブルクリック
2インポートウィンドウが出たら「Import All」を押す
3完了後、Project画面にファイルが並んでいればOK
4Scenes フォルダを開き、.unityファイルをダブルクリックしてシーンを開く
M-05
Unity画面の見方（最低限これだけ）
PENDING
▼

Unityの画面は最初は複雑に見えるが、改変に使うエリアは実質4〜5か所だけ。全部覚えなくていい。

Sceneワールドの3D空間を見る画面。直接オブジェクトを動かせる
Hier.ワールドにある全オブジェクトの一覧。ここから選択する
Insp.選んだオブジェクトの詳細設定。位置・サイズ・素材などを変更する
Proj.素材・テクスチャ・音楽などのファイル置き場
Con.エラーや警告が表示される。赤いエラーが出たらここで確認
M-06
基本の改変をやってみよう
PENDING
▼

まずは「小さな変更を1つ加えてみる」ことが目標。完璧にしようとせず、動かしてみることが大事。

1追加：Hierarchyで右クリック → 3D Object → Cube などを選ぶ
2移動・回転・拡大：Scene画面上部のツール（W / E / R）で操作する
3削除：Hierarchyで選択 → Delete キー
4保存：Ctrl+S で必ず保存。こまめに保存する癖をつけよう
M-07
VRChatにアップロードしよう
PENDING
▼

改変が終わったら、VRChatに自分のワールドとしてアップロードする。初回はVRChatアカウントへのログインが必要。

1上メニュー「VRChat SDK」→「Show Control Panel」を開く
2VRChatアカウントでログインする
3「Builder」タブ → 「Build & Publish for Windows」をクリック
4ワールド名・説明を入力して「Upload」。アップロード完了まで待つ
5VRChatを起動し、Worldsで自分のワールドを検索して入れたら成功！
M-08
うまくいかないときは？
PENDING
▼

詰まったときのよくあるパターンと対処法。必要なときだけ読めばOK。

Unityが開かない・クラッシュする
Unityのバージョンが合っているか確認。VCCから開き直すと安定することが多い。
オブジェクトがピンク色になる
マテリアルが見つかっていない状態。Projectからマテリアルを探してInspectorにドラッグすると直ることが多い。
アップロードボタンが押せない
Consoleに赤いエラーがないか確認。エラーがあるとアップロードできない。エラー文をコピーして検索すると解決策が見つかる。
VRChatに反映されない
アップロード完了後、VRChatを完全に再起動してみよう。反映まで数分かかることもある。
SKILL BOOK — やりたいことから探す
S-01
アイテムを持ち上げたい
TOUCH EASY
▼
WHAT

プレイヤーがオブジェクトを手で拾い上げ、持ち歩いたり投げたりできる。

USES

VRC Pickup コンポーネント（VRChat SDK に入っている）

STEPS
1持ち上げたいオブジェクトをHierarchyで選択する
2InspectorでAdd Component → 「VRC Pickup」を追加する
3Rigidbodyが自動で追加される。そのままでOK
4アップロードして確認。オブジェクトの上でトリガーを引くと持てる
⚠ Rigidbodyの「Is Kinematic」にチェックが入っていると持てないことがある。外してみよう。
S-02
ドアを開けたい
TOUCH NORMAL
▼
WHAT

クリック（Interact）するとドアが開閉するアニメーションが動く。

USES

Animator（Unity標準）+ VRC Trigger / Udon（SDK）

STEPS
1ドアのオブジェクトにAnimatorをAdd Componentする
2Animation Clipを2つ作る（Open状態・Close状態）
3AnimatorのTransitionに「open」「close」パラメータを設定する
4UdonSharpでInteractイベントにパラメータを切り替えるスクリプトを書く
⚠ 最初は「最も単純な1方向に開くドア」だけを目標にしよう。同期は後回しでOK。
S-03
椅子に座れるようにしたい
TOUCH EASY
▼
WHAT

オブジェクトに近づいてトリガーを引くと、プレイヤーが着座ポーズになる。

USES

VRC Station コンポーネント（VRChat SDK に入っている）

STEPS
1椅子（または空のGameObject）にAdd Component → 「VRC Station」を追加
2「Seated」にチェックを入れる（座りポーズを使う場合）
3Player MobilizerをNoneのままでもOK。アップロードして確認
⚠ 座る位置がズレる場合は、Stationオブジェクトの位置・回転を調整しよう。
S-04
物をすり抜けさせたい（貫通）
TOUCH EASY
▼
WHAT

「壁として機能する当たり判定」をなくし、プレイヤーや物体がすり抜けられるようになる。

USES

Collider の Is Trigger チェックボックス

STEPS
1対象オブジェクトをHierarchyで選択する
2InspectorでCollider（Box ColliderやMesh Collider）を探す
3「Is Trigger」にチェックを入れる → これだけで当たり判定がなくなる
⚠ Is Triggerをつけると「物体が乗れなくなる」ので、床には絶対につけないこと。
S-05
見えない壁・床を作りたい
TOUCH EASY
▼
WHAT

目には見えないが、プレイヤーが通れない透明な壁・床・天井を作れる。

USES

Unityの Cube/Plane + Mesh Renderer を無効化

STEPS
1Hierarchy → 右クリック → 3D Object → Cube を作る
2Sceneでサイズ・位置を壁の形に合わせる
3InspectorのMesh Renderer左のチェックを外す → 見えなくなる
4Box Colliderは残っているので当たり判定だけが残る
S-06
近づいたら音を鳴らしたい
LOOK EASY
▼
WHAT

近づくと音楽・SE・環境音が聴こえる。距離で音量が変わる3D音響も設定できる。

USES

Unityの Audio Source コンポーネント + 音声ファイル（MP3/WAV）

STEPS
1音を出したい位置にGameObjectを置く（空でOK）
2Add Component → 「Audio Source」を追加
3「AudioClip」に音声ファイルをドラッグ、「Play On Awake」にチェック
4Spatial Blend を 1 にすると3D音響（距離で音量変化）になる
S-07
霧・空気感・全体の雰囲気を変えたい
LOOK EASY
▼
WHAT

霧の濃さ・色・距離を変えたり、スカイボックスを変えてワールドの雰囲気をガラッと変えられる。

USES

Window → Rendering → Lighting → Environmentタブ

STEPS
1Unityの上メニュー「Window」→「Rendering」→「Lighting」を開く
2Environmentタブで「Skybox Material」を変更して空を変える
3「Other Settings」→「Fog」にチェックを入れて霧をONにする
4Fog ColorとDensityで霧の色・濃さを調整する
S-08
オブジェクトに光を当てたい
LOOK EASY
▼
WHAT

任意の場所に光源を置いてワールドを明るくしたり演出できる。

USES

Unityの Light コンポーネント（Point / Spot / Directional）

STEPS
1Hierarchy → 右クリック → Light → Point Light（または Spot Light）を選ぶ
2Sceneで光らせたい位置に移動させる
3InspectorでColor・Intensity（明るさ）・Range（届く距離）を調整
⚠ ライトを増やしすぎるとパフォーマンスが下がる。Point Lightは10個以内を目安に。
S-09
特定の場所にワープしたい
MOVE NORMAL
▼
WHAT

特定のオブジェクトやエリアに触れると、別の場所へ瞬間移動できる。

USES

VRC Station（着地型）または UdonSharp で Teleport スクリプト

STEPS — VRC Station方式
1ワープ先にGameObjectを置き、VRC Stationを追加する
2「Seated」のチェックを外す（立ったまま移動させる場合）
3ワープ元のオブジェクトにTriggerコライダー（Is Trigger ON）を設置
4触れた瞬間にStationに乗せる処理をUdonで記述する
⚠ Udonの基礎知識が少し必要。まずは「物を持つ」「すり抜け」などを終えてから挑戦しよう。
